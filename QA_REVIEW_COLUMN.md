# QA Review — COLUMN (Index / Detail)

対象: `/column` / `/column/[slug]`

## 優先度
COLUMNは「読み物」だが、入口（一覧）と読了後の次アクション（回遊）が弱いと離脱が起きやすい。
P0（タップ/フォーカス）と、P1（読み物の型：要点→目次→章ジャンプ）を先に揃える。

## 赤入れ（改修前に起きやすい問題）
1) 詳細ページに目次が無く、章ジャンプができない
- 長文での“探索コスト”が高く、途中離脱が起きやすい

2) 一覧のチップ/ページ番号/前後移動が小さくなりがち
- フィルタUIが“押せるが押しにくい”状態になりやすい

3) 読了後に「上へ戻る」が弱い
- スクロールが深い記事ほど、次の行動が雑になる

## 実施した改修（規格適用）
- COLUMN詳細に `InThisStoryToc`（目次）を追加
  - H2（CHAPTER）を抽出して自動生成
  - `sticky` で追従し、長文でも章ジャンプを最短化
- PILLAR導線 / 読了CTA / TOPへ戻る を `.cb-tap`（44px基準）で統一
- COLUMN一覧のチップ/ページネーション/主要CTAに `.cb-tap` を付与
  - “当たり判定”は拡張しつつ、見た目の雰囲気は維持

## 変更ファイル
- `app/column/page.tsx`
- `app/column/[slug]/page.tsx`
- `components/content/InThisStoryToc.tsx`（目次のタップ&フォーカス強化は全ページへ波及）
- `app/globals.css`（Checklistラベルのタップ基準）

## 目視で最終確認すべき項目（Vercel Preview）
- iPhone幅で、`/column` のカテゴリ/タグ/ページ送りが片手で誤タップなく回れるか
- `/column/[slug]` で「要点→目次→章ジャンプ→TOPへ戻る→関連記事→一覧」が迷わず回れるか
- 目次のフォーカス（Tab）表示がカードの外に欠けず“確実に見える”か
