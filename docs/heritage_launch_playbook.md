# HERITAGE 初速（Discover / SNS）運用プレイブック

## 目的
- 「HERITAGEを読みに来る」動機を作る（指名流入の芽を増やす）
- 野良流入（Discover / SNS）を入口に、CARS → COLUMN → GUIDESへ自然に回遊させる
- 投稿の型を固定して、毎回同じ密度とトーンで量産できる状態にする

---

## 1) Discoverを取りに行くための“投稿の作法”
### A. 見出し（タイトル）の型（日本語）
- 1行目で「何の話か」が分かる（年・車名・論点のうち2つは入れる）
- あおり語を避ける（最強／神／ヤバい 等）
- 40〜55文字を目安（短く言い切る）

**例**
- 1995年、速さが“再現性”に変わった。R33とDC2で読む分岐点
- 1999年、90年代JDMが完成形に見えた理由。S2000／S15／R34
- 2002年、速さの前提が変わった。Z33とDC5が作った新しい基準

### B. サムネ/OGPの型
- 1枚で統一（ブランドとしての“顔”を固定）
- 文字を載せるなら最小限（年号＋短いキーワード）
- 余白を多めにして、車が中央に来る構図が安定

---

## 2) X（旧Twitter）投稿テンプレ
### A. 1投目（フック＋結論）
```
1995年のR33とDC2は、速さの競争を「再現できる体験」に変えた2台でした。
派手さより、崩れない状態・反応の明確さを作る方向へ。

全文（HERITAGE）：
{URL}
```

### B. 2投目（ポイント3つ）
```
この記事の要点：
1) R33は長い負荷で崩れない“再現性”
2) DC2は量産の中に“反応の明確さ”を残した
3) 共通点は、速さを説明できる状態に整えたこと

{URL}
```

### C. 3投目（回遊の入口）
```
この章で出てきた車はCARSにまとめています（画像＋スペック＋関連記事）。
まずは車から入るのもアリ。

CARS：
{CARS_URL}
```

**運用メモ**
- 1記事につき、初日は3投（上記A→B→C）で固定
- 翌日以降は「1投＋引用リポスト」で再露出

## 3) Instagram（フィード/リール/ストーリー）テンプレ
### A. フィードキャプション（短め）
```
1999年は、90年代JDMが“完成形”に見えた年。
S2000(AP1)／S15／R34／Evo VI。
速さを誇るより先に、速さの理由が伝わる状態に整っていた。

全文はHERITAGEへ：
{URL}

#carboutiquejournal #JDM #S2000 #Silvia #GTR
```

### B. ストーリー（3枚構成）
- 1枚目：年号＋テーマ（例「1999：完成形」）
- 2枚目：車名を3つだけ（S2000 / S15 / R34）
- 3枚目：リンク＋一言（「速さの理由が分かる」）

---

## 4) 1週間の初速プラン（固定で回す）
- Day1：新規HERITAGE公開 → X 3投 → IG ストーリー
- Day2：記事内の“1セクション”だけ切り出して再投稿（X 1投 / IG 1投稿）
- Day3：CARSへの導線を前面に出す（車ページへ誘導）
- Day4：COLUMNを絡める（背景知識の補足として）
- Day5：GUIDESを絡める（「維持」「チェック」「選び方」はGUIDES側へ）
- Day6：再掲（反応が良かった投稿を軽く手直しして再投）
- Day7：まとめ（“今週のHERITAGE”として3本を並べる）

---

## 5) 回遊を増やすための書き方チェック（HERITAGE）
- 1セクションにつき「言い切りの結論」→「理由」→「設計/現場の話」→「読後に残る一文」
- 車名は出しっぱなしにせず、必ず“その車らしさ”の根拠（構造/狙い）まで書く
- 余計な比喩、ことわざ、決めつけの断言を避ける（読者の反発ポイントになりやすい）
