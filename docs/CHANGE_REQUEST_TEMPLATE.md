# docs/CHANGE_REQUEST_TEMPLATE

変更依頼（または作業開始時のメモ）テンプレ。

## 目的
- 何を良くするか / 何を追加するか

## 現状
- どのページ/コンポーネント/データが関係するか

## 制約（破らないこと）
- `docs/DESIGN_CONSTRAINTS.md` の該当条項
- 既存機能を落とさない

## 変更内容（最小単位で箇条書き）
- 例: components/X を追加
- 例: lib/viewmodel/Y に整形関数追加
- 例: data/Z.json に項目追記

## 影響範囲
- 対象URL
- 影響するデータ

## 検証
- `npm run validate`
- `npm run build`
